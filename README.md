# 우리학교 학사일정 캘린더

학생과 학부모가 학교 학사일정을 쉽게 확인할 수 있도록 만든 캘린더형 웹앱입니다.

## 주요 기능

- 학교명 검색
- 학교 선택 및 선택 학교 저장
- 월별 학사일정 캘린더
- 이번 달 주요 일정 리스트
- 방학/시험/행사 빠른 필터
- 일정 키워드 검색
- 모바일 반응형 화면
- 나이스 교육정보 개방 API 연동 준비 구조

## 2-1단계 변경사항

- 딥블루 중심 + 보라 포인트 컬러로 조정
- NEIS API 직접 호출이 아닌 프록시 API 연동 준비
- API 키를 프론트엔드에 노출하지 않는 구조로 정리
- 학교 검색 API와 학사일정 API 함수 분리
- mock 데이터 기반 fallback 구조 유지
- 학교 검색/학사일정 로딩 상태 추가
- 오류 상태 안내 UI 추가
- 선택 학교 localStorage 저장
- 새로고침 후에도 선택 학교 유지
- 월 이동 시 일정 재조회 구조 적용

## 향후 프록시 API 구조

```text
GET /api/schools?keyword=학교명
GET /api/schedules?officeCode=B10&schoolCode=7010000&year=2026&month=6
```

## 향후 개발 예정

- Render 또는 Vercel 프록시 서버 구축
- 나이스 학교기본정보 API 연동
- 나이스 학사일정 API 연동
- 학년별 일정 필터
- 카카오톡 공유용 일정 복사
- 급식/시간표 API 확장 검토

## 배포

GitHub Pages에 정적 웹앱으로 배포할 수 있습니다.


## 2-1 보정

- 니블 스타일의 보라색 중심 컬러로 복원
- 딥블루 버튼 색상을 보라/퍼플 포인트로 조정
- 히어로 그라데이션은 장식 없이 부드러운 보라→블루 흐름으로 개선

## 2-1 색감 보정

- 니블스 계열 보라 포인트 컬러 복원
- 배경의 보라/파랑 반짝임 제거
- 히어로 그라데이션을 보라 중심으로 차분하게 조정
- 검색/오늘/선택 버튼을 보라 계열로 통일


## 2-1 색감 재보정

- API 연동 준비 구조는 유지하고, 1.9 버전의 차분한 니블스 계열 컬러로 복원했습니다.
- 검색 버튼, 선택 강조색, 배경 광, 히어로 그라데이션을 과하지 않은 보라/파랑 톤으로 조정했습니다.


## 2-1 색감 재정리

- 2-1단계 API 준비 구조는 유지하되, 화면 색감은 1.9 버전의 차분한 니블스 계열 컬러로 복원했습니다.
- 과한 보라 강조색과 배경 광을 제거하고, 기존 보라-블루 그라데이션과 버튼 톤을 유지합니다.


## 2-3단계: Render 프록시 연결

이번 버전은 mock 데이터가 아니라 Render 프록시 서버를 통해 NEIS API와 연결됩니다.

```js
const API_CONFIG = {
  useMock: false,
  baseUrl: "https://school-calendar-proxy.onrender.com"
};
```

연결 구조:

```text
GitHub Pages 프론트엔드
↓
Render Flask 프록시
↓
NEIS Open API
```

테스트 순서:

1. 프록시 서버 확인: `https://school-calendar-proxy.onrender.com/health`
2. 학교 검색 확인: `https://school-calendar-proxy.onrender.com/api/schools?keyword=서울`
3. 프론트에서 학교명을 검색하고 실제 학교 목록이 표시되는지 확인
4. 학교를 선택한 뒤 월별 학사일정이 표시되는지 확인

주의: Render 무료 플랜은 처음 접속 시 서버가 잠들어 있다가 깨어나느라 30초 이상 걸릴 수 있습니다.


## 2-4단계: 지역 선택 검색 개선

전국 학교 검색 시 같은 학교명이 여러 지역에 있을 수 있어 검색 흐름을 개선했습니다.

- 시도교육청 선택 후 학교명 검색
- `/api/schools?keyword=학교명&officeCode=교육청코드` 구조 지원
- 검색 결과 개수 표시
- 검색 결과가 많을 때 지역 선택 안내
- 검색 결과 카드에 지역 배지와 학교급 배지 표시
- 선택 버튼 문구를 `이 학교 선택`으로 변경
- 선택한 학교명을 상단바에 표시
- 상단바 `학교변경` 버튼 추가


## 2-5단계: 달력/주요 일정 가독성 개선

- 달력 칸에서 말줄임 처리된 일정명에 마우스를 올리면 전체 일정명이 표시됩니다.
- 노란 안내 박스를 달력 바로 아래로 이동했습니다.
- 주요 학사일정 목록 카드의 날짜, 일정명, 설명, 유형 배치와 여백을 개선했습니다.
- 모바일에서도 주요 일정 카드가 더 안정적으로 보이도록 조정했습니다.


## 2-6단계: 일정 카드 문구 정리

- 학사일정 목록에서 세부 내용이 없는 경우 `세부 내용은 학교 안내를 확인해 주세요.` 문구를 표시하지 않도록 정리했습니다.
- 학교 검색 예시를 `학돌중`, `서울학돌초` 기준으로 변경했습니다.


## 2-7단계: 학부모용 사용성 개선

- 달력에서 일정이 있는 날짜를 클릭하면 해당 날짜 일정만 모아 볼 수 있습니다.
- 선택한 날짜 일정 보기 패널과 `전체 일정 보기` 버튼을 추가했습니다.
- Render 무료 서버 첫 요청 지연을 안내하는 부드러운 안내 박스를 추가했습니다.
- 검색 결과 카드의 학교명, 학교급, 지역, 주소 위계를 개선했습니다.
- 학교 선택 후 검색 결과는 자동으로 접히고, 선택한 학교 일정 영역으로 이동합니다.
- 상단바의 학교명을 클릭하면 현재 선택한 학교 카드로 이동합니다.


## 2-8단계: 달력 hover 상세내용 개선

- 달력 칸의 일정 pill에 마우스를 올리면 일정명뿐 아니라 세부 내용까지 함께 표시됩니다.
- 날짜 칸에 마우스를 올리면 해당 날짜의 일정 목록을 줄바꿈 형태로 확인할 수 있습니다.


## 2-9단계: 실제 배포 안정화/완성도 정리

- 모바일에서 일정 있는 날짜에 `눌러보기` 안내를 추가했습니다.
- 오류 발생 시 `다시 시도` 버튼을 추가했습니다.
- Render 무료 서버 첫 요청 지연 상황을 더 자연스럽게 안내합니다.
- 검색 결과가 많을 때 상위 일부만 표시될 수 있다는 안내를 추가했습니다.
- 선택한 학교의 월별 일정 공유 문구 복사 기능을 추가했습니다.
- 선택 날짜 패널의 안내 문구를 개선했습니다.


## 2-9.1단계: 날짜 클릭 동작 수정

- 달력에서 일정 있는 날짜를 클릭하면 해당 날짜 일정만 아래 주요 일정 영역에 표시되도록 이벤트 처리를 보강했습니다.
- 날짜 칸 키보드 접근성도 함께 보완했습니다.
- `눌러보기` 안내 문구를 `클릭 시 그날 일정`으로 정리했습니다.
